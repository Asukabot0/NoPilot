# 003 - discover 应先采集用户 idea 再展开调研

## 来源

`.claude/commands/discover.md` Step 0 — Constraint Collection（第14-29行）

## 问题描述

当前流程：Step 0 直接展示约束清单让用户填写 → 推荐模式 → Layer 1 方向选择。

但实际场景中，用户是带着一个碎片化的 idea 来的（可能只是一句话、一个灵感），当前流程没有给用户表达 idea 的入口，直接就问技术栈、时间约束等具体问题，体验很割裂。

## 期望行为

1. 先问用户"你有什么想法？"，接受自由输入（一句话、一段描述、甚至只是一个关键词）
2. 对碎片化 idea 进行整理、归纳、补全
3. 再基于整理后的 idea 有针对性地收集约束（而非无脑列出 6 个问题）
4. 然后才进入竞品调研和方向选择

## 改进方向

- Step 0 拆分为：Idea 采集 → Idea 整理 → 针对性约束收集
- 约束收集可以根据 idea 内容智能决定哪些约束需要问（而非全部列出）
- 与 issue 001（约束清单一次性提问过多）关联，一并改进
