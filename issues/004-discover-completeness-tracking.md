# 004 - discover 每轮对话应评估各环节完整度和澄清度

## 来源

`.claude/commands/discover.md` 全流程（Layer 1 / Layer 2 / Layer 3）

## 问题描述

当前 discover 流程是线性推进的，用户说 APPROVE 就往下走，没有机制评估当前阶段的讨论是否真的足够充分。这导致：

- 某些关键细节可能被跳过，到 spec 阶段才发现信息不足
- 用户可能过早 APPROVE，遗漏重要约束或边界条件
- 没有量化指标告诉用户"哪些部分还不够清晰"

## 期望行为

每轮对话结束时，自动评估并展示各环节的完整度/澄清度，例如：

```
当前讨论完整度：
├── 核心功能定义  ████████░░  80%  ✓ 基本清晰
├── 用户场景覆盖  ██████░░░░  60%  ⚠ 缺少异常流程
├── 技术约束      ████░░░░░░  40%  ✗ 数据存储方案未讨论
├── 边界条件      ██░░░░░░░░  20%  ✗ 并发/性能要求未明确
└── 非功能需求    ░░░░░░░░░░   0%  ✗ 未触及
```

当各项达到阈值后才建议进入下一阶段，避免 spec 阶段信息不足。

## 改进方向

- 设计一个完整度评估算法，覆盖维度可包括：功能定义、用户场景、技术约束、边界条件、非功能需求等
- 每轮对话后自动运行评估，输出可视化进度
- 设定进入下一阶段的最低完整度阈值（可配置）
- 对不足的维度自动生成追问建议，引导用户补充
