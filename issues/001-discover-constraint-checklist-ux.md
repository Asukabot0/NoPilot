# 001 - discover skill 约束清单一次性提问过多

## 来源

`.claude/commands/discover.md` Step 0 — Constraint Collection + Mode Recommendation（第14-29行）

## 问题描述

discover skill 在初始阶段一次性列出 6 个约束问题让用户回答：

1. 技术栈限制
2. 时间约束
3. 目标平台
4. 明确排除项
5. 预算/资源约束
6. 现有资产

这种交互方式对用户不友好，信息量过大，容易让用户感到压迫感。

## 改进方向

- 考虑渐进式提问，逐个或分组询问
- 或者只问最关键的 1-2 个问题，其余设为可选/后续补充
- 或者根据用户描述的内容自动推断部分约束，只确认不确定的部分
